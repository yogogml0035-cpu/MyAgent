# 03 标准答案

## 自测题答案

1. API 和 Runner 测试依赖 fake storage。如果 fake 比生产更宽松，测试会通过，但生产会失败。
2. 流式 Agent 会产生很多中间信号。追加事件能保留全过程，也能让前端断线后补齐；覆盖文本会丢历史。
3. 删除 task 如果只删数据库，不删工作区，会留下上传文件和产物，造成隐私风险和磁盘垃圾。
4. expected statuses 可以防止并发或重复写入把已完成、已取消、已失败的任务错误改回其他状态。

## 练习观察点

未知 `after_id` 返回完整事件流是一个恢复策略。前端再用事件 ID 合并去重，就能兼容旧标签页、断线重连和事件 ID 迁移。
