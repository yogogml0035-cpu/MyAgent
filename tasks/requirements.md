# 前端 Skill 选择器业务 PRD 对齐稿

## 目标
- 让用户在聊天输入框中通过 `/` 快速查看并选择当前项目可用 skill，选择后以 Codex 风格 chip 进入输入框，方便继续补充需求并发送。

## 范围
- 包含：展示当前项目 `backend/skills` 下所有可用 skill、输入 `/` 触发列表、按关键词筛选、点击或键盘选择、选中后生成 chip。
- 不包含：Codex 全局 skill、用户本机其他 skill 目录、skill 创建/编辑/删除、skill 版本管理。

## 业务场景

场景1：输入 `/` 打开 Skill 列表
- 用户在聊天输入框输入 `/` 时，前端展示当前项目 `backend/skills` 下所有可用 skill。
- 列表展示 skill 名称和简介，不展示本地路径或敏感运行信息。
- `/` 后继续输入关键词时，列表按 skill 名称和简介过滤。

场景2：选择 Skill 后生成 Chip
- 用户点击 skill 后，该 skill 以 Codex 风格 chip 插入输入框。
- chip 可被用户识别为已选择 skill，而不是普通文本。
- 插入后焦点回到输入框，用户可以继续输入自然语言需求。
- 选择 skill 不自动发送消息，仍由用户主动点击发送。

场景3：关闭与空状态
- 用户按 `Escape`、点击外部、删除触发内容或发送消息后，skill 列表关闭。
- 无匹配 skill 时展示轻量空状态。
- skill 列表加载失败时，聊天输入仍可正常使用。

## 验收口径
- 输入 `/` 后能看到 `backend/skills` 下所有可用 skill。
- 输入 `/关键词` 后能筛选 skill。
- 点击 skill 后输入框出现 Codex 风格 chip。
- chip 和后续用户输入一起保留在待发送内容中。
- 页面不展示 skill 文件路径。
- 上传、模型选择、发送、停止任务等现有能力不受影响。

## 需要对齐的问题
- 无。
