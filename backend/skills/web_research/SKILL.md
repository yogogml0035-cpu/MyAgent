# 联网研究技能

## 工作流程

1. 如果用户上传了文件，先用资源工具读取必要内容，提取搜索所需的关键参数。
2. 使用 `searxng_search` 工具做针对性搜索。
3. 分析搜索结果，判断哪些证据直接支持或削弱结论。
4. 综合发现并组织成连贯回答，在可能的情况下注明来源。

## 边界

- 这是快速联网核查技能，不是深度研究代理。
- 总搜索调用不超过 5 次；每次 `max_results` 不超过 5。
- 不要使用 `task`/sub-agent 委派，不要展开多代理研究流程。
- 不要生成 Word、PPT、Excel、PDF、HTML 或其他报告产物，除非用户明确要求可下载文件。
- 搜到足够证据后立即回答；如果搜索引擎返回验证码、502、timeout、无结果或明显无关结果，说明证据不足和不确定性，不要继续循环搜索。

## 使用提示

- 使用更具体的搜索词以获得更好的结果
- `searxng_search` 的 `engine` 参数只允许 `bing` 或 `baidu`
- 每次搜索根据任务需要选择一个引擎：英文/国际网页优先 `engine="bing"`，中文网页可尝试 `engine="baidu"`
- `trust_env` 默认 `false`；如果搜索返回网络错误、502 或 timeout，可尝试 `trust_env=true` 走系统代理环境重试
- 对“是否倾向性/指向性”这类核查，优先搜索产品参数、政府采购公告、厂商型号和竞品参数；结论应区分“高度疑似”“有一定指向性”“证据不足”。
